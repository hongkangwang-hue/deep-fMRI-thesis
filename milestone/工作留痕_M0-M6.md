# M2-C — English1000 参照验证（已完成）

**目标**：证明数据加载 + 时间对齐 + 评分忠实于 LeBel 参考产物 `results/eng1000/UTS03/corrs.npz`。

## 纪律（已预注册冻结于 `frozen/m2c_reference_validation.yaml`）

指标/种子/容差在查看结果前冻结。Phase 1 不通过先定位差异来源，不放宽容差。

## Phase 1 = PASS（硬闸门，服务器 GPU）

计划文档原打算用 CPU 原生 bootstrap_ridge 重算；**实际执行改为服务器 GPU 独立 PyTorch
float64 重实现**（`scripts/m2c_phase1_gpu.py`，非改 `ridge.py`），输出
`results/eng1000_gpu_rerun/UTS03/`。

**四道硬闸门全部通过**：

| 指标 | 阈值 | 实测 |
|------|------|------|
| voxel_r | ≥0.995 | **0.9962** |
| median\|Δr\| | ≤0.01 | **0.0000** |
| p95\|Δr\| | ≤0.05 | **0.0103** |
| valpha 同档 | ≥0.85 | **0.9980** |

mean_r=0.0407, max_r=0.6980（参考 ~0.041/0.70）。

**踩坑链（已全修）**：`norm(0)` 是 0-范数非 L2 → float32 系统偏差（必须 float64）→
float64 OOM（Rresp 按 bootstrap 传输 + alpha 按体素分块）→ 最终 corrs 须用
`sqrt(|R²|)·sign(R²)` 非 Pearson r（对应 `use_corr=False, return_wt=True`）→
**`torch.var`/`std` 默认 `ddof=1` 须改 `ddof=0` 匹配 numpy**（最后 ddof 修复让 mean
0.092→0.041）。

## Phase 2 = CHECK/接受（描述性，非硬闸门，服务器 GPU）

用冻结 spec（PCA-100 + himalaya GPU + 3 折故事级 CV）跑 eng1000，与 native 比较。

- 空间相关 vs native = **0.7498**（阈值≥0.80，spec 标注"低于则排查，不是硬闸门"）。
- ROI 符号正确（left_IFG=0.2012, bilateral_PT=0.2390，均正）。
- 泄漏检查 PASS。

**已排查归因**（非代码 bug）：① alpha 网格不同（native `[10,1000]` vs 冻结 spec
`[0.01,10000]`）；② alpha 选择方式不同（native 50×bootstrap 平均 vs 冻结 spec 仅
inner 2-fold 单次，噪声更大）；③ PCA-100 截断（保留 92.59% 方差，次要因素）；
④ solver 实现不同（native 手写 SVD `ridge_corr` vs himalaya 内置 random-search）。
支持性证据：冻结 spec mean_r（0.1069）反而**高于** native（0.0407），符合"更少平滑
正则化→单体素 r 更高但空间模式与 native 不同"的解释。**用户决策**：接受为可归因的
描述性偏差，记入 `results/eng1000_phase2_frozen/UTS03/run_manifest.json::investigation`，
不再深挖，进入 M3。

**himalaya 踩坑（已修）**：`RidgeCV` 默认全量 95556 体素 + 13 alpha 摊平算导致 18GB
单次分配 OOM → 加 `solver_params(n_targets_batch=5000, n_targets_batch_refit=2000,
n_alphas_batch=5, local_alpha=True)`；`Y_in_cpu` 是顶层参数不能塞进 `solver_params`；
`torch_cuda` 后端下 `model.predict()`/`best_alphas_` 返回 CUDA tensor，须用
`backend.to_numpy()` 转换。

# M3a — 盲态参数核查（已完成）

**目标**：只用 outer training + inner CV，不碰 held-out r，检查 λ 边界/PCA/solver/
内存是否需要在正式竖切前调整。

**脚本**：`scripts/m3a_blind_check.py`（按 inner-CV score 分层看信号体素边界命中率，
全脑命中率无意义，因大量体素本就无信号）。

**发现**：冻结网格 `logspace(-2,4,13)=[0.01,1e4]` 下，**信号体素**（inner-CV
score>0.10，n≈15000，cv_max=0.508）**100% 撞上界 1e4**（不是噪声假警报）。
根因：PCA-100 输出的 PC scores 未标准化（sklearn `whiten=False`），方差远大于
native 的单位方差特征，所需正则化强度大几个数量级。

**决策 = 方案A**（扩网格，非 PCA whiten 方案B）：扩到 `logspace(-2,7,19)=[0.01,1e7]`
后，信号体素（n≈18000-20000，cv~0.5）边界命中率降至 **0.000**（上下界都0，3折一致，
完美落回网格内）。

**重新冻结**：`analysis_spec.yaml` + `config.yaml` + `pipeline.py::LAMBDA_GRID`
统一改为 `logspace(-2,7,19)`（commit `10204f1`，tag `m3a-lambda-refreeze`）。

# M3 — 评分方法学重构（已完成）

**目标**：`pipeline.py::run_fold` 原实现拼接所有测试故事 TR 后算一次 Pearson r，
不符合冻结文档模块4"先对每个 held-out story 单独计算 voxel r → Fisher-z → ROI，
再形成 fold 级汇总"的要求（拼接会引入类 Simpson 悖论的偏差）。

**修复**（commit `edea295` 拼接 bug + `1f93646` 完整重构）：`score.py` 加
`roi_mean_fisherz`/`weighted_mean_scalar`；`pipeline.py` 加 `StoryScore` dataclass，
`run_fold` 改为逐 story 切片算 voxel r → 每 ROI fisher-z 聚合 → `effective_tr_weighted_mean`
按有效 TR 数汇总为 fold 级；`FoldResult`/`CVResult` 保留 `story_scores`（M4/M5 的
per-story 结果需求）。15 个单测全过（含 3 个新测试证明拼接 vs 逐段加权数学上确实不同）。

**验证**：pythia/H128/UTS03/3折 重跑，新网格+新评分下 voxel_r mean=0.0655（旧
0.0557），ROI left_IFG=0.1322（旧0.1179）bilateral_PT=0.1390（旧0.1216）——数值全部
提升，符合"修复正则化+修复评分偏差都应改善拟合质量"的预期。λ 撞上界比例
16.6%/15.7%/16.4%（3折）与 M3a 全脑诊断的 19.1% 一致（无信号噪声体素的正常现象）。

## 上服务器前的逻辑复查抓到一个真 bug

原加载器假设 `n_eff_tr` 在所有 layer/condition 下全局一致，但 M4 主层（shift 限制
mask）与最终层 normal（非限制 mask）的 n_eff **本来就不同**——旧断言在真实数据上
会直接崩溃。已修复为：权重按 key 存（每条件用自己 mask 的 n_eff），配对靠"所有
key 共享同一组 story 重抽索引"实现，与权重是否相同无关；共同 mask 校验只在该成立
处做（主层 normal==shift，对应验收标准5）。commit `df33018`，本地用假数据（主层
n_eff=100系列、最终层n_eff=120系列 + 故意破坏共同mask两种场景）验证修复正确后
才上服务器跑真实数据。

### 稳健性（层位翻转判定：主层 vs 最终层）

- **mamba_minus_pythia**：主层CI侧="+"，最终层CI侧="+"，方向一致，**结论稳健**。
- **rwkv_minus_pythia**：主层CI侧="-"（显著），**最终层CI跨0（不再显著）**。未达
  "实质翻转"标准（需两层CI各自非零且反号），但显著性**依赖层位选择**——论文中

## ⚠️⚠️ 重大方法学修正：40s 负控制的分层解读（用户审图发现）

初版 Figure 4 标题写"shifted removes effect (as expected)"，**这是 overclaim**。
用户指出：图上 shifted 条件下 Pythia/Mamba/RWKV 的 Δr_total 依然是正的，"效应消除"
这个说法站不住。深入分析后发现这是三个不同层面混在一起报告：

1. **绝对 r**：被摧毁（IFG r 从~0.13降到~0.007-0.011）——干净通过。
2. **架构间 Δr_total 差值（确认性家族本体）**：平移后塌陷、不再排除0——**干净通过，
   两项确认性结论(Mamba>Pythia, RWKV<Pythia)是词序特异的，站得住**。
3. **每个模型自己的原始 Context Gain**：**之前从未计算过这个配对差值**。真实数据
   算出来：pythia -0.0004[-0.0029,+0.0020]（CI跨0）；mamba +0.0023[-0.0004,+0.0051]
   （CI跨0）；**rwkv -0.0039[-0.0063,-0.0015]（paired CI位于0下方，即shifted>normal，
   比"没削弱"更反常）**；awd_lstm~0。**结论：单模型的原始Δr_total不能解读为长程
   语言整合的证据**——这不动摇两项确认性结论（层面2），但严格限制了"Mamba的
   Context Gain最大"这类单模型描述性数字的解读力度。

**代码修复**（commit `9a7652c`→`24f4417`→`2a37f7c`）：
- `src/stats/estimands.py` 新增配对差值估计量 `delta_total_normal_minus_shift_

## mask 一致性验证升级：从"计数相等"到"逐元素相同"

用户进一步指出：M5 loader 之前只断言 normal/shift 的 **n_eff（有效TR计数）相等**，
这是**必要非充分**条件——两个 mask 完全可能选中不同 TR 却计数相同。修复
（commit `2a37f7c`，**为未来数据集设计，非一次性验证**）：

- `src/ridge/pipeline.py`：`StoryScore` 新增 `scoring_mask` 字段，存实际布尔评分 mask。
- `src/ridge/m4_driver.py`：**源头断言**——以后任何数据集跑 M4，都会自动逐 story
  对比 normal/shift 的布尔 mask，`np.array_equal` 不通过直接 `raise`；不再用
  n_eff 计数代替。
- `scripts/verify_scoring_mask_identity.py`（新增，可复用）：① 证明 `apply_fir`/
  `shift_story_no_wrap` 的 valid 输出**与特征值无关**（只依赖位置/长度）——这是
  "重构的mask等于M4当时用的mask"的前提，用不同随机值验证两次输出完全相同；
  ② 用 frozen respdict 的真实故事长度重构 normal/shift 布尔 mask，逐元素断言相同。
  UTS03 上验证：83/83 held-out story mask 完全相同，74 种故事长度特征无关性全部
  成立。**服务器和本地各自独立跑过一次，结果一致**（交叉验证）。
- `tests/test_m2_alignment.py` 新增 2 个单测锁定"mask与特征值无关"这个前提（不依赖
  数据，进CI）。81 个测试全过。
