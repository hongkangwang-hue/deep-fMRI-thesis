# 里程碑总览：三被试执行计划（对齐 V6.3，事实核查修订版）

## 本文档的地位

本文档是三被试扩展实验唯一的正式执行依据，取代此前围绕 UTS03 单被试案例撰写的 `assignment1.md` 在三被试范围内的效力。`assignment1.md` 记录的是 UTS03 pilot 的真实执行过程，继续作为 UTS03 那一段历史的可追溯记录保留，但不再是三被试阶段的规划来源——凡本文档与 `assignment1.md` 在三被试相关设计上不一致之处，以本文档为准，并应将本文档已冻结的规则（尤其是 M5 的跨被试统计口径）同步补写回 `assignment1.md` 或其后继文档，避免团队内出现两份互相矛盾的规划记录。

## 当前真实数据状态（写入本文档前已核实）

本轮修订前对本机数据与 Claude Code 审计结果做了核实，如实记录如下，作为 M0/M1 排期的现实约束，不是方法学假设：

- 已下载并完整验证：UTS03（M0–M6 全流程已跑通，达成单被试毕业闭环）。
- 尚未下载：UTS01、UTS02 的 BOLD derivatives。这是 M1 无法立即启动的直接原因，必须先完成下载才能进入 M1 的 BOLD 接入与 ROI 构建。

## V6.2 相对 V6.1 的修订说明

本版不改变任何研究设计，只修正两处此前表述不足、可能被后续实现误读的执行细节，并据此补充相应的验收条款。

1. **normal 与 40 秒 time-shift 必须独立重拟合。** 两条件从 TR 级语言特征处分叉后，各自独立拟合 training-only scaler、PCA、λ 与 Ridge，不得复用 normal 已拟合的对象，也不得只对 normal 的 prediction 做平移。两条件只在共同评分 mask 上取交集比较，拟合过程本身互不共享。这一点此前只强调了"评分 mask 逐元素相同"，未明确拟合环节本身要分开，容易被误实现为"拟合一次、平移预测"。
2. **English1000 参照验证复用 UTS03 的参考协议本身，不套用正式三折/PCA-100 主实验路径；UTS03 观测到的 0.9962 是该被试的验证结果，不是新被试必须达到的统一阈值。** 硬闸门检验的是"是否忠实复现同一参考路径"，不是被试信号强弱，因此不因新被试相关较低而放宽实现容差。
3. 同时补充两处形式化澄清：Ridge 的拟合键为 `subject × fold × model × H × layer × condition`，ROI 不进入拟合键、只在评分阶段选列；正式矩阵中 `最终层 × shift40` 明确列为非必需格子，不默认纳入毕业闭环。



## 项目定位

本项目执行 V6.1 计划书：在统一的 voxelwise Ridge 读出器下，比较四个序列模型在 H ∈ {8, 32, 128} 三个上下文长度下的 brain alignment 与 Context Gain。核心比较组为 Pythia-160M、RWKV-4-Pile-169M、Mamba-130M；fastai AWD-LSTM（WT103_FWD，约 44M）作为经典 LSTM 历史参照，不参与核心三模型的强架构排名。数据集为 LeBel ds003020（v3.1.1，commit cb7c536d），被试为 UTS01、UTS02、UTS03。ROI 为左 IFG（主要）与双侧 PT / planum temporale（参照）。

主表示层为约 2/3 相对深度的标准化主层；最终层仅作三名被试 × IFG 的预设稳健性核查。Context Gain 拆为 Δr_local（r32−r8）、Δr_long（r128−r32）、Δr_total（r128−r8），主指标为 Δr_total。

本项目为被试内设计。三名被试逐一独立建模、逐一独立报告；语言模型特征与 eligible_h128 目标集合只取决于刺激文本，三被试共享、只提取一次；BOLD 侧的一切逐被试单独完成。三名被试不合并为人群样本，不做跨被试池化；主证据形式是同一架构效应在三名被试身上是否方向一致地重现。

本文档与 V4.9 里程碑总览的根本区别在于执行现实：UTS03 的 M0–M6 全流程已完整跑通并达成过一次单被试毕业闭环，构成本项目的已验证基线。因此本里程碑不再从零搭建管线，而是把已验证的 UTS03 管线锚定为方法基准，逐被试扩展到 UTS01、UTS02，再做跨被试综合与论文整合。

## 执行逻辑：复用已验证基线，逐被试扩展，跨被试综合

整条路径不按计划书章节顺序执行，而按"已验证 → 逐被试复用 → 跨被试综合"的依赖执行。

M0 锚定已验证的 UTS03 基线与其冻结配置，并把三被试扩展所需的全部规则（单一 eligible_h128 目标集合、λ 网格、word midpoint 口径、方法 B 聚合、逐被试统计口径、已知偏差）写入扩展冻结文件。

M1 与 M2 是逐被试的 BOLD 侧去风险轨道：M1 为 UTS01、UTS02 各自接入 BOLD、皮层表面并构建 voxel mask 与 ROI 列索引；M2 为每名新被试单独通过 English1000 参照验证硬闸门。语言模型特征与 eligible_h128 不在此重做，直接复用 UTS03 缓存。

M3 用一名新被试跑一条最小端到端竖切，确认复用的管线在新被试上行为正常（λ 无大量边界命中、维度、mask、time-shift 诊断）。M3 通过后 M4 才对 UTS01、UTS02 各自扩展到完整正常与 time-shift 全矩阵。M5 只做逐被试预注册对比、置信区间、跨被试一致性判读，不在统计阶段重新拟合模型。M6 完成三被试图表与论文整合。

## 三个信心检查点

1. M0：UTS03 已验证基线的冻结配置已锚定；三被试扩展规则（目标集合、λ、midpoint、方法 B、逐被试统计、已知偏差）已写入扩展冻结文件并打 tag；UTS01、UTS02 数据可得性已确认。
2. M2：UTS01、UTS02 各自的 voxel mask 与 ROI 列索引构建完成，且各自通过 English1000 参照验证硬闸门，数值一致性达标。
3. M3：某一新被试的最小端到端竖切生成维度正确、无泄漏的 held-out 结果；λ 网格在新被试上无大量边界命中；normal 与 shift 评分 mask 逐元素相同。

三个检查点全部通过后，后续主要是逐被试矩阵扩展、统计和跨被试综合，不应再引入新的方法逻辑。

## 任务优先级与工期取舍

所有任务按以下三级管理：

- **P0｜实验有效性底线：必须完成。** 删除后可能造成跨被试口径不一致、训练—测试泄漏、上下文失控、故事边界污染或主问题无法回答。P0 未完成，不得进入下一里程碑。
- **P1｜完整硕士论文交付：原则上完成。** 用于形成完整三被试结果、稳健性分析和规范论文材料；工期风险时在保住全部 P0 后压缩 P1 的展示范围，而不是改变主分析。
- **P2｜发表级可复现性加分：工期紧时可简化。** 包括逐文件 SHA256、每次临时运行的完整 manifest、扩展审计索引。K=50 敏感性检查目前没有任何冻结规则可执行，不计入本轮 P2 范围；若要做，须先补写一条包含比较对象、判定标准的独立规则（见 M0 配置示例中的 `k_sensitivity_spotcheck` 项），补写后再决定是否列入 P2。

排期默认优先完成 P0 和必要 P1。M7 及 P2 不得阻塞三被试主实验闭环。

## 毕业闭环定义

原则上的毕业闭环为：完成 M0–M6，覆盖 UTS01、UTS02、UTS03 三名被试（逐被试独立建模与报告）、四模型、H={8,32,128}、标准化主层、左 IFG 与双侧 PT、三折故事级 CV、training-only scaler/PCA、logspace(−2,7,19) 网格的统一 voxelwise Ridge、方法 B 聚合、三类 Context Gain、H-specific 架构差值、逐被试配对 story bootstrap、逐 H 的 40 秒 time-shift 与其三层解读、IFG 最终层稳健性核查、>100s 共同评价 mask，以及三名被试的跨被试一致性描述。

单被试（UTS03）的结果虽已存在，但毕业闭环要求三名被试逐被试完成上述流程并做跨被试综合。确认性结论按被试分别成立，跨被试只做描述性一致性，不做池化。

AWD-LSTM 是计划内的正式历史参照，不得在执行过程中默认删除；若无法稳定接入，须形成书面 scope amendment 并经导师确认。M7（LPP）、Narratives、native state、完整 CCnorm、全皮层均不影响毕业闭环。

## 有效性纪律由结构强制执行

fold 划分、ROI 规则、上下文定义、层位、评价 mask、Ridge 选择规则、PCA 规则、time-shift 规则、方法 B 聚合口径和统计 contrasts 均在 UTS03 上已冻结，扩展到新被试时不得更改，只允许在每名被试各自的数据内重新拟合 scaler/PCA/Ridge。任何为新被试放宽规则的行为都视为破坏三被试同一管线。最低审计证据（P0）为：

- Git commit 与 freeze tag；
- 扩展冻结文件 hash；
- 数据集版本与 story manifest；
- 模型 revision / commit；
- 可重建的运行环境 lock file；
- 每名被试正式运行的 run manifest。

UTS01、UTS02 各自的第一次 brain score 前，其 voxel mask、ROI 列索引须已冻结，English1000 闸门须已通过。

## 预注册的已知偏差

为使执行与论文口径一致，预先声明三处已知的、方向明确的偏差与边界，均不威胁两项确认性结论的效度，写入 Methods 与 Limitations，均不需要为纠正而重跑已冻结的 UTS03。

1. 内层预处理的轻度乐观：λ 的内层 2-fold 选择使用在整个 outer-training 上拟合的 Scaler 与 PCA，内层 validation 折参与了 PCA 估计。该偏差不触及 outer held-out、只影响每体素 λ 选择，且在四模型同向的差值估计中高度对消。
2. 训练端特征支撑：训练 mask 使用 FIR_valid（与 H 无关）而未额外施加 100 秒规则，H=128 每个故事前段存在近零的退化训练行，方向上使 Context Gain 偏保守（低估而非虚增）。评分端已实证公平：全部 84 个故事 H=128 首词 midpoint 均早于 100 秒。
3. repeatability 可得性：若重复故事仅提供跨重复平均而无逐次响应，repeatability 不可得，此时 PT 一律保守表述为未检测到可靠效应，不作区域特异性支持。

# 里程碑 0：扩展冻结与 UTS03 基线锚定

## 周期与验收要求

- 周期：第 1 周。起止日期：待锚定。
- 前置依赖：无（UTS03 已验证基线已存在）。
- 验收要求：在进入 M1 前完成。
- 验收方式：扩展配置检查 + UTS03 基线冻结物核对 + 数据可得性检查 + contrast registry 走查 + Git tag/hash 检查。

## 阶段目标

把三被试扩展所需的全部"查看新被试结果前必须确定"的规则固定下来，并把 UTS03 已验证基线锚定为方法基准。本阶段不产生新被试的 brain score，不重新提取语言特征。

完成后，应固定并可追溯：

- UTS03 已验证基线的冻结配置与代码 tag，作为三被试同一管线的基准；
- 被试范围 UTS01/UTS02/UTS03 与各自数据可得性确认；
- 单一 eligible_h128 目标集合定义（刺激侧、与模型输出有效性无关）；
- λ 网格 logspace(−2,7,19)、word midpoint 时间戳、方法 B 聚合口径；
- 逐被试统计口径与确认性 Holm 家族的 per-subject scope；
- 三处已知偏差的书面声明；
- 三折故事级 CV、>100s、FIR、time-shift 规则（沿用 UTS03）；
- 预注册 contrasts 与其被试维度声明。

### 本阶段优先级

- **P0：**扩展冻结文件、UTS03 基线 tag 锚定、eligible_h128 定义、λ/midpoint/方法 B 口径、逐被试统计与 contrast 分层、已知偏差声明、Git freeze tag。
- **P1：**UTS01/UTS02 数据可得性报告、目录规范、逐被试 run manifest 模板。
- **P2：**逐文件 SHA256、完整 hash 索引、所有临时运行 manifest。

## 设计思路

所有参数从一个扩展主配置入口读取，不得散落魔数。UTS03 已验证基线的冻结物（配置、frozen/ 索引、contrast registry、代码 tag）作为只读基准被引用，不复制、不修改；扩展只新增被试侧的可变部分。三被试共享同一份语言特征缓存与 eligible_h128 目标集合，这些是刺激侧事实，在 M0 确认其 hash 与 UTS03 一致后即冻结复用。

单一目标集合 eligible_h128 由 0-based 索引 i ≥ 128（首个合法目标为 i=128，其前方恰有索引 0–127 的 128 个历史词）与词—时间标注质量检查唯一定义，不查询任何模型的输出有效性。因此 AWD-LSTM 的词表覆盖不可能缩小核心三模型可用的样本；AWD-LSTM 的 xxunk 只作元数据记录，从不用于排除目标词；AWD-LSTM 被排除出确认性结论仅通过 contrast registry 的家族声明实现。

4. 逐被试 QC 报告与冻结物 hash。

# 里程碑 2：新被试 English1000 参照验证闸门

## 周期与验收要求

- 周期：第 2 周。起止日期：待锚定。
- 前置依赖：M1。
- 验收要求：每名新被试各自通过 English1000 参照验证硬闸门后，方可进入 M3。
- 验收方式：数值一致性硬闸门检查 + 时间对齐与评分忠实性核对。

## 阶段目标

对 UTS01、UTS02 各自跑一次 English1000 参照验证，确认数据加载、word→TR 时间对齐（按 word midpoint）、FIR 与评分在新被试上同样忠实于官方/已验证参考产物。这是新被试进入正式评分前的去风险硬闸门，对应 UTS03 上已通过的同一验证。

### 本阶段优先级

- **P0：**每名新被试 English1000 参照验证达标；闸门协议与正式主实验路径的分离。
- **P1：**验证报告与差异定位记录。
- **P2：**扩展诊断图。

## 设计思路

English1000 参照验证是一个与四个正式语言模型无关的独立闸门，且**协议本身已完整冻结、不需要为新被试重新记录**：训练/测试 session 切分（sessions 1–5，测试故事为 wheretheressmoke）、求解器设置（alphas=logspace(1,3,10)、nboots=50、chunklen=40、nchunks=125、trim=5、ndelays=4、pca=none、固定种子）与四道容差阈值（voxel_r≥0.995、median|Δr|≤0.01、p95|Δr|≤0.05、valpha 同档≥0.85）均已写入代码库的冻结配置与求解脚本说明中，且这些阈值是与被试无关的通用常数，不是为 UTS03 单独调过的——因此"复用协议本身"这条可以在 UTS01、UTS02 上直接执行，不需要先补写协议记录。

真正待核实的只有一件事，而且是数据可得性问题，不是协议记录问题：**UTS01、UTS02 是否存在对应的参照真值文件**（即 UTS03 使用的那一份 `corrs.npz` 或等价物的新被试版本）。这份参照真值本身的来源在代码库里也没有完整记录（冻结配置的注释写明其原始生成过程未知，是外部预先存在的产物），因此无法从协议或代码推断新被试是否有对应版本，必须在 M2 正式开始前逐个核实这两名被试是否存在该文件；若不存在，需要先确定能否用同一条已冻结的参考路径重新生成，或改用其它已验证的等价参照，并把这个决定记录下来，而不是默认假设文件存在直接开跑。

闸门检验的是"新被试的数据加载、midpoint 对齐、Lanczos 重采样、FIR 与评分链条是否忠实复现同一已记录协议"，而不是另建一套看起来相似的新基线，也不是任何架构结论。UTS03 在该闸门上观测到的 voxel 相关 0.9962 是一个已完成验证的观测结果，**不直接写成 UTS01、UTS02 必须达到的统一数值阈值**——真正要达到的是上面那四道已冻结的容差阈值本身；不得因为新被试的科学信号（brain alignment）本身较低就放宽这四道阈值，硬闸门比较的是实现一致性，不是被试的 brain score 高低。

建议比较层级从粗到细：参考协议中的训练/测试故事或 session 顺序与有效 TR 数；参考设计矩阵与响应矩阵形状；故事边界、裁剪和 FIR 边界；voxel-level prediction；voxel r 向量；与参照真值文件的相关或误差。ROI 汇总只作辅助定位，不替代 voxel-level 的主判据。未通过时先定位差异来源（BIDS 结构、TR、ROI 空间、midpoint 计算、故事/session 顺序错位），不放宽四道容差阈值、不进入正式评分。

## 模块与目的

1. 参照特征装配模块
   目的：为每名新被试装配 English1000 特征并按 midpoint 对齐到 TR，装配对象为 UTS03 硬闸门协议自身冻结的训练/测试组织，不擅自替换为正式三折划分。

2. 参照评分模块
   目的：使用该参考协议冻结的求解器、正则化选择与评分规则跑一遍，与官方/已验证参考产物比对 voxel-level prediction 与 voxel r；ROI 汇总仅作辅助定位。

3. 差异定位模块
   目的：未达标时按 story、TR、voxel、fold 分层查找首个偏差来源。

## 交付物

1. UTS01、UTS02 各自的参照真值文件（`corrs.npz` 或等价物）存在性核实结果；若缺失，记录补生成方案或替代方案。
2. UTS01、UTS02 各自的 English1000 参照验证报告与达标结论，逐项对照四道冻结阈值。
3. 与参照真值的 voxel-level prediction、voxel r 一致性数值。
4. UTS03 观测值 0.9962 的登记说明（标注为观测值，非跨被试阈值）。
5. 未达标项的差异定位记录（如有）。

## 任务拆解

1. 核实参照真值文件
   - 逐名新被试检查是否存在对应的 `corrs.npz` 或等价参照真值；缺失则先确定补生成或替代方案并记录，不默认假设存在。

2. 装配参照特征
   - 为每名新被试生成 English1000 特征，按 midpoint 对齐；训练/测试组织沿用已冻结的协议本身（sessions 1–5、测试故事 wheretheressmoke），不默认套用正式三折。

3. 跑参照评分
   - 使用已冻结的求解器设置（alphas/nboots/chunklen/nchunks/trim/ndelays/pca/种子），比对 voxel-level prediction 与 voxel r；ROI 汇总仅作辅助定位。

4. 判定闸门
   - 逐项核对四道冻结阈值（voxel_r≥0.995、median|Δr|≤0.01、p95|Δr|≤0.05、valpha 同档≥0.85）；未通过则定位差异、修正数据/对齐，不因新被试信号弱而放宽任一阈值。

## 验收标准

1. UTS01、UTS02 各自的参照真值文件存在性已核实，缺失项已有补生成或替代方案。
2. UTS01、UTS02 各自通过 English1000 参照验证硬闸门，逐项达到四道已冻结的容差阈值。
3. 验证协议为代码库中已冻结的协议本身，未被替换为正式三折/PCA-100 主实验路径。
4. UTS03 的 0.9962 已登记为观测值，未被写成或用作 UTS01/UTS02 的统一通过阈值。
5. 时间对齐使用 word midpoint，与 UTS03 一致。
6. 比较以 voxel-level 结果为主判据，ROI 汇总仅作辅助定位。
7. 未达标项均有差异定位与修正记录，未通过闸门者不进入正式评分。

## 明确不做

1. 不为通过闸门而放宽四道已冻结的容差阈值，也不因新被试信号弱而放宽。
2. 不用四个正式语言模型替代 English1000 做本验证。
3. 不默认把正式主实验的三折/PCA-100 路径套进本闸门协议。
4. 不把 UTS03 的 0.9962 当作 UTS01/UTS02 必须达到的统一阈值。
5. 不在未核实参照真值文件是否存在的情况下默认开跑。
6. 不跳过任一新被试的闸门直接进入正式矩阵。
7. 不用正式模型结果反向证明参照验证"应该没问题"。

- 验收方式：竖切结果走查 + λ 边界命中检查 + 共同 mask 逐元素检查 + time-shift 诊断。

## 阶段目标

在一名新被试（建议 UTS01）上跑一条最小端到端竖切，确认复用的、已在 UTS03 验证过的下游管线在新被试的 BOLD 上行为正常。竖切覆盖四模型 × H={8,32,128} × 标准化主层 × outer fold 1 × normal 与 shift 两条件，不铺全部三折。

本阶段是"迁移正确性"闸门，不是提前做统计结论或架构比较。诊断只允许回答：新被试的数据是否进入正确的模型矩阵；λ 网格在新被试上是否仍覆盖合理范围；PCA-100 是否可拟合且信息保留可记录（evr@100）；normal 与 shift 是否使用完全相同的最终评分 mask；shift 是否为故事内非循环平移；结果是否有限、非全零、非与另一条件完全相同；输出字段是否与 UTS03 一致。诊断禁止回答：哪个模型最好；新被试是否复制了 UTS03 的绝对数值；是否需要修改 H、层位或 ROI；是否应删除某个低分条件。看到本阶段的 held-out r 不构成修改任何冻结规则的理由，除非确证是实现错误。

### 本阶段优先级

- **P0：**竖切的无泄漏、维度、λ 边界、normal/shift 独立拟合与 mask 一致性核查。
- **P1：**evr@100、竖切诊断记录。
- **P2：**扩展诊断输出。

## 设计思路

竖切复核的目的是在铺开昂贵的全矩阵之前，先证明"复用管线 × 新被试 BOLD"这一组合没有隐性问题，而不是抢先下科学结论。

normal 与 shift 是两条独立拟合路径，从此处开始就要按这个结构跑：先得到按 word midpoint、Lanczos 对齐到 TR 的语言特征；shift 路径在每个故事内对该 TR 级特征做 40 秒非循环平移（边缘置无效），再对 normal 与 shift 各自独立地在其 outer-training stories 上拟合 training-only scaler、PCA-100、内层 λ 选择与 Ridge，分别生成各自的 held-out prediction——不得复用 normal 已拟合的对象，也不得只对 normal 的 prediction 做平移。两条独立路径最终在预先构建的共同评分 mask 上比较。

重点核查四件事：其一，λ 网格 logspace(−2,7,19) 在新被试上是否仍无大量边界命中（若信号体素大量撞界，须在查看更多结果前排查，而非直接扩网格）；其二，normal 与 shift 是否各自独立完成了拟合、且最终评分 mask 是否逐元素相同（np.array_equal 断言，不一致直接 raise）；其三，两条件各自的 evr@100 是否可记录；其四，held-out r 量级是否正常（不要求高，但不应全为负值或异常偏离 UTS03 量级，否则提示新被试的 BOLD 接入或对齐有误）。单个条件的 shifted r 未明显低于 normal，不构成本阶段的硬失败——竖切只用一折数据，噪声较大；正式的负控制结论留到 M4 全矩阵与 M5 统计判断，不在竖切阶段抢先下结论。

## 模块与目的

1. 竖切执行模块
   目的：在新被试、主层、fold 1、四模型上跑 normal + shift 最小切片。

2. 独立拟合模块
   目的：分别为 normal、shift 路径拟合各自的 training-only scaler/PCA/λ/Ridge，不共享拟合对象。

3. 边界与 mask 核查模块
   目的：检查 λ 边界命中率、evr@100、normal/shift 最终 mask 逐元素一致性。

4. 量级诊断模块
   目的：核对 held-out r 量级、有限性；不据模型间高低做任何方法调整。

## 交付物

1. 新被试竖切的 held-out r（主层、fold 1、四模型、normal + shift）。
2. λ 边界命中率报告与两条件各自的 evr@100。
3. normal 与 shift 独立拟合对象的追溯记录（scaler/PCA/λ/Ridge 各自来源）。
4. normal 与 shift 最终评分 mask 逐元素一致性断言结果。
5. 竖切诊断记录（工程通过/不通过判定，不含架构结论）。

## 任务拆解

1. 跑竖切
   - 新被试、主层、fold 1、四模型，normal 与 shift。

2. 独立拟合 normal 与 shift
   - 分别在各自 outer-training stories 上拟合 scaler/PCA-100/λ/Ridge，记录各自 evr@100；不复用彼此对象。

3. 核查 λ 与 mask
   - 检查两条件各自的边界命中；逐元素断言最终评分 mask 相同。

4. 量级诊断
   - 核对 held-out r 量级与有限性；单折 shift 未明显低于 normal 不算硬失败，记录留待 M4/M5 判断。

## 验收标准

1. 竖切生成维度正确、无窗口外历史、无跨故事泄漏的 held-out 结果。
2. λ 网格在新被试上无大量边界命中，或已在看更多结果前排查。
3. normal 与 shift 已各自独立完成 scaler/PCA/λ/Ridge 拟合，未复用彼此对象、未仅平移 prediction。
4. normal 与 shift 最终评分 mask 逐元素相同。
5. held-out r 与 story score 均为有限值；单折 shift r 未明显下降不构成本阶段失败，记录后留待 M4/M5 整体判断。
6. 未根据模型间排名修改任何冻结参数或提前停止某一模型。

## 明确不做

1. 不在竖切阶段铺全矩阵或跑 bootstrap/Holm。
2. 不因竖切 r 不理想而回头修改冻结规则或 λ 网格（除非确证实现错误）。
3. 不跳过 mask 逐元素核查。
4. 不在本阶段比较模型排名或据此判断架构优劣。
5. 不把单折 shift r 未明显降低当作项目失败直接推翻设计。

# 里程碑 4：逐被试全矩阵扩展

## 周期与验收要求

- 周期：第 3–4 周。起止日期：待锚定。
- 前置依赖：M3。
- 验收要求：UTS01、UTS02 各自的正常矩阵、time-shift 矩阵与最终层 IFG 矩阵完整。
- 验收方式：矩阵完整性检查 + 共同 mask 检查 + evr 记录检查 + run manifest 追溯。

## 阶段目标

在 UTS01、UTS02 上各自跑满四模型 × 三上下文 × 三折的主层正常矩阵、40 秒 shift 矩阵与最终层 IFG 矩阵。语言特征与 eligible_h128 直接复用 UTS03 缓存；正式拟合单元的键为 `subject × outer_fold × model × H × layer × condition`，ROI 不进入该键、只在评分阶段选列；同一键下对完整有效 voxel 矩阵拟合一次。

并非该键的全部组合都需要覆盖，正式矩阵的必需与非必需格子预先划定：

- 主层 × normal：必需；
- 主层 × shift40：必需；
- 最终层 IFG × normal：必需；
- 最终层 IFG × shift40：不属于 V6.1/V6.2 必需矩阵，若要执行须作为单独登记的 P2 项，不默认纳入毕业闭环。

### 本阶段优先级

- **P0：**每名新被试主层正常 + shift + 最终层 IFG 全矩阵、normal/shift 独立拟合、共同 mask、无泄漏。
- **P1：**evr@100 记录、λ summary、逐被试 run manifest、断点续跑与失败恢复。
- **P2：**最终层 shift40（如执行，须单独登记）；K=50 敏感性检查未纳入本轮范围（无冻结规则，见 M0 配置示例说明）。

## 设计思路

每名新被试的全矩阵在其各自 BOLD 上独立生成：模型表示缓存跨 fold、跨被试、跨 condition 复用；scaler、PCA、FIR、Ridge 与预测是 fold-dependent、subject-dependent，且 **condition-dependent**——normal 与 shift40 在同一个 (subject,model,H,layer,fold) 下是拟合键中两个不同的 condition 取值，各自独立拟合，不共享已拟合对象，也不能通过平移 normal 的 prediction 得到 shift 结果（具体拟合流程见 M3 已验证的独立拟合模式，本阶段只是把它扩展到全部 fold）。每个 held-out story 保存独立预测、真实 BOLD、有效 mask 与 story-level score，并标注所属被试与 condition，以支持 M5 逐被试的配对 story bootstrap。聚合按方法 B：逐 story voxel r → Fisher-z → ROI 内平均 → 按有效 TR 在 z 空间加权到 fold 级与被试级 → tanh 回 r。为核查固定 K=100 的架构公平性，每个 (model,H,fold,condition) 记录 PCA 在 K=100 的累计解释方差比 evr@100。

time-shift 逐故事独立、不回卷、退化位置置无效；两条件各自拟合完成后，其最终评分 mask 须逐元素相同（`after_100s ∩ FIR_valid ∩ shift_valid` 的交集对两条件一致）。特征装配对非有限值设防，避免单模型的非有限输出经共享设计矩阵污染其它模型。

由于全矩阵计算量较大、跨多个 fold/model/H/condition 单元，执行须支持断点续跑：任务调度只为尚缺失的单元生成计算任务，不重复已通过且已冻结的 fold；中断后从 condition/fold 级别继续，不覆盖已验证结果。矩阵覆盖表记录每个格子的状态（完成/失败/跳过/不适用），任何缺失单元必须有明确归类，不允许静默缺失。

## 模块与目的

1. 正常矩阵模块
   目的：四模型 × 三上下文 × 三折主层 held-out 预测与评分。

2. time-shift 矩阵模块
   目的：逐故事 40 秒 shift，独立拟合，最终 mask 与 normal 逐元素一致。

3. 最终层模块
   目的：IFG 最终层四模型 × 三上下文 × 三折 normal 矩阵（shift40 为可选 P2）。

4. evr 与 manifest 模块
   目的：记录 evr@100、λ 选择、run manifest。

5. 矩阵调度与故障恢复模块
   目的：根据缺失单元生成任务，不重复已通过的格子；中断后从 condition/fold 级继续；维护矩阵覆盖表。

## 交付物

1. UTS01、UTS02 各自的主层正常、40 秒 shift、最终层 IFG 全矩阵。
2. 每个 held-out story 的独立预测、truth、mask、有效 TR、score，标注所属被试与 condition。
3. 每个 (model,H,fold,condition) 的 evr@100 与 λ 选择记录；normal/shift 拟合对象可区分且均可追溯。
4. 矩阵覆盖表：每个格子的完成/失败/跳过/不适用状态。
5. 逐被试 run manifest 与失败恢复日志。

## 任务拆解

1. 跑正常矩阵
   - UTS01、UTS02 各自四模型 × 三上下文 × 三折主层，复用特征、逐被试逐 fold 拟合下游。

2. 跑 shift 矩阵
   - 逐故事 40 秒 shift；对每个 (model,H,layer,fold) 独立拟合 shift 条件的 scaler/PCA/λ/Ridge，不复用 normal 对象；最终评分 mask 与 normal 逐元素核验。

3. 跑最终层 IFG 矩阵
   - 四模型 × 三上下文 × 三折 normal；shift40 不在本阶段必需范围内。

4. 记录 evr 与 manifest
   - 存每个 (model,H,fold,condition) 的 evr@100、λ summary、run manifest。K=50 抽查未纳入本轮范围，不在此步执行。

5. 矩阵覆盖检查与断点续跑
   - 从已通过的格子继续补齐缺失单元，不重跑已验证格子；任何缺失明确归类为失败/跳过/不适用。

## 验收标准

1. UTS01、UTS02 各自主层正常、shift、最终层 IFG normal 矩阵完整（4×3×3）。
2. 特征与 eligible_h128 复用 UTS03 缓存（同一 hash）；scaler/PCA/Ridge 逐被试逐 fold 单独拟合，且 normal 与 shift 作为拟合键中不同的 condition 各自独立拟合，未共享已拟合对象。
3. normal 与 shift 最终评分 mask 逐元素相同；无跨故事 FIR、无跨故事 shift、无跨 fold 拼接评分。
4. 每个 (model,H,fold,condition) 已记录 evr@100，normal/shift 拟合来源可追溯区分。
5. 矩阵覆盖表无未解释缺口；断点续跑未覆盖已验证结果。
6. 所有结果可由逐被试 run manifest 追溯。

## 明确不做

1. 不为新被试重新提取语言特征或重核 eligible_h128。
2. 不跨被试共享 scaler/PCA/Ridge 拟合，也不在 normal 与 shift 之间共享。
3. 不用 np.roll 或填零后评分，不通过平移 normal 的 prediction 得到 shift 结果。
4. 不把最终层 shift40 设为毕业闭环必需；K=50 敏感性检查因无冻结规则，本轮不执行。
5. 不在本阶段做统计显著性解释或据 r 删除被试、故事、体素或 ROI。

# 里程碑 5：逐被试统计与跨被试综合

## 周期与验收要求

- 周期：第 5 周。起止日期：待锚定。
- 前置依赖：M4。
- 验收要求：三名被试各自的预注册对比、CI 与确认性结论完成，跨被试一致性判读完成。
- 验收方式：contrast registry 执行核查 + 逐被试 bootstrap 检查 + 跨被试一致性走查。

## 阶段目标

对三名被试各自执行预注册对比、配对 story bootstrap 与确认性 Holm 校正，再做跨被试一致性的描述性综合。统计逐被试独立，不跨被试池化。

### 本阶段优先级

- **P0：**逐被试两项确认性 Δr_total 架构对比 + Holm 校正、逐被试配对 bootstrap、跨被试一致性判读、time-shift 三层解读。
- **P1：**RQ1 H-specific 差值、Δr_local/long、PT 描述、层位稳健性、evr 报告。
- **P2：**扩展稳健性分析。K=50 敏感性方向一致性因无冻结规则，本轮不执行（若 M4 补写了规则并执行，此处再补相应的方向一致性判读）。

## 设计思路

统计严格分被试也严格分层。每名被试各自跑配对 story bootstrap（fold 内故事重抽、四模型共享同一批 story indices、方法 B 汇总、种子固定、1000 次），得到各自的 r、Δr、核心架构差值与 95% CI。两项 IFG 主层 Δr_total 架构差值为确认性家族，按被试分别做 Holm 校正——每名被试一个 2 对比家族，分别下确认性结论，不合并成跨被试大家族。其余全部为探索性/描述性/稳健性。

跨被试只做描述性判读：报告 RWKV−Pythia、Mamba−Pythia 的 Δr_total 差值方向与量级是否在三名被试身上一致重现。三者方向一致且各自 CI 不跨 0 为强的被试内重复；方向不一致如实报告为被试间异质性。不生成合并后的组水平 CI。

time-shift 分三层报告：绝对 r（应大幅下降）、架构间 Δr_total 差值（CI 应塌陷、不再排除 0）、每模型 normal 减 shifted 的配对差值（诊断性、未校正、非确认性，中性措辞）。

## 模块与目的

1. 逐被试 bootstrap 模块
   目的：每名被试独立重抽，方法 B 汇总，输出点估计与 CI。

2. 确认性校正模块
   目的：每名被试两项 Δr_total 架构差值的 Holm 校正与结论。

3. 跨被试综合模块
   目的：方向一致性的描述性判读，不池化。

4. 负控制与层位模块
   目的：time-shift 三层解读、主层/最终层实质翻转判定。

## 交付物

1. 每名被试、每 ROI、每模型的 r8/r32/r128 与 95% CI。
2. 每名被试每个 H 下 RWKV−Pythia、Mamba−Pythia 的 r 差值与 CI。
3. 每名被试三类 Context Gain 与 CI。
4. 每名被试两项确认性 Δr_total 架构差值、CI、bootstrap p 值与 Holm 结果。
5. 跨被试方向一致性描述表（一致 / 部分一致 / 不一致）。
6. 正常与 shifted 的 r、三类 Δr；time-shift 三层解读结果。
7. IFG 主层与最终层预注册对比及实质翻转判定。
8. IFG/PT repeatability（若可得）与 ROI 解释表；evr@100 报告。

## 任务拆解

1. 逐被试故事级结果核查
   - 每名被试每个故事在所有成对条件下均存在；核查共同 mask 与权重。

2. 逐被试执行 contrasts
   - RQ1 H-specific 差值；三类 Δr；两项确认性 Δr_total 架构差值。

3. 逐被试配对 bootstrap
   - fold 内重抽、四模型共享 indices、方法 B、1000 次、固定种子。

4. 逐被试确认性校正
   - 每名被试 2 对比家族做 Holm；分别下确认性结论。

5. 跨被试综合
   - 描述性判读方向一致性；不池化、不生成组水平 CI。

6. 负控制与层位
   - time-shift 三层解读；主层/最终层实质翻转按相反非零 CI 判定。

## 验收标准

1. 全部估计逐被试独立完成；bootstrap 不跨被试重抽，不把 voxel/TR/被试当作可交换独立样本。
2. 确认性 Holm 家族按被试分别校正；三名被试各自得到独立确认性结论与 CI，未合并成组水平家族。
3. 跨被试仅作方向一致性的描述性判读，未生成组水平 CI。
4. 聚合口径统一为方法 B；差值在 r 空间取。
5. time-shift 分绝对 r、架构差值、每模型配对差值三层报告；第三层标注为诊断性、未校正、非确认性。
6. 层位翻转按相反非零 CI 判定，不以点估计排名替代。
7. PT 解释结合 repeatability、基础 r、CI 宽度与 shift 结果；repeatability 不可得时表述为未检测到可靠效应。

## 明确不做

1. 不把三名被试合并成人群样本做池化或组水平推断。
2. 不把三被试的确认性对比合并成单一 Holm 家族。
3. 不在 bootstrap 中重跑语言模型、PCA 或 Ridge。
4. 不把 AWD-LSTM 绝对 r 纳入核心三模型强排名。
5. 不把每模型 normal 减 shifted 配对差值当作确认性证据。

# 里程碑 6：三被试图表、论文整合与可复现交付

## 周期与验收要求

- 周期：第 6 周。起止日期：待锚定。
- 前置依赖：M5。
- 验收要求：完成即构成毕业闭环。
- 验收方式：图表走查 + RQ1–RQ4 对应检查 + Methods/Results 一致性检查 + 可复现包检查。

## 阶段目标

生成三被试论文级图表，整合各阶段 Methods，完成 Results 与结论边界，交付可复现的冻结配置、运行清单和结果文件。

## 设计思路

五图两表与 RQ 一一对应，全部逐被试并排呈现，不给合并组水平数值。AWD-LSTM 使用虚线/灰度/单独子图，图注注明其训练语料、参数量、tokenizer 与 hidden width 与核心组不完全可比，只解释自身上下文曲线。跨被试判读用"该效应在三名被试中方向一致 / 部分一致 / 不一致"这类描述性语言，主证据是被试内效应的跨被试重复性而非合并显著性。

Methods 须与代码口径完全一致：写明 word midpoint、方法 B 聚合、单一 eligible_h128、逐被试统计与三处已知偏差。结论边界保持：approximately matched checkpoint-level evidence、相对深度为操作性近似、三个 H 为粗粒度采样、三名被试支持被试内重复而非人群外推、voxelwise Ridge 是统一读出器。

## 验收标准

1. 五图两表与 QC 表完整，每张图/表对应明确 RQ 或控制目的，且逐被试呈现。
2. Results 通过直接 contrasts 回答 RQ1–RQ4，跨被试用一致性描述而非合并显著性。
3. Methods 与代码实现、配置、run manifest 一致，含 midpoint、方法 B、单一 eligible_h128、逐被试统计与已知偏差。
4. AWD-LSTM 仅作历史参照，图表与文字均未用于核心强排名。
5. 只使用采样点措辞，不声称精确饱和点或函数形式。
6. 结论限定为 checkpoint-level、被试内、间接机制证据；不做跨被试池化推断。
7. 可复现包能定位每项结果的 Git tag、配置 hash、数据 hash、模型 revision 与运行命令。

## 明确不做

1. 不在 M6 重新选择分析规则或统计 contrasts。
2. 不根据图表回头修改冻结的 fold、ROI、层位、mask 或 λ。
3. 不把三名被试合并成组水平结论或用合并 CI 替代逐被试呈现。
4. 不让 LPP/Narratives 等可选分析阻塞毕业闭环。
5. 不用 AWD-LSTM 绝对 r 进行核心架构排名。
