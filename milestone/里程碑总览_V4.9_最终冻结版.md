# 里程碑总览：上下文长度与序列架构的语言脑编码比较（对齐 V4.9，最终冻结版）

## 项目定位

本项目实现并执行 V4.9 计划书的目标 A：在统一的 voxelwise Ridge 读出器下，比较四个序列模型在 H ∈ {8, 32, 128} 三个上下文长度下的 brain alignment 与 Context Gain。核心比较组为 Pythia-160M、RWKV-4-Pile-169M、Mamba-130M；fastai AWD-LSTM（WT103_FWD，约 44M）作为经典 LSTM 历史参照，不参与核心三模型的强架构排名。主数据集为 LeBel extended（UTS01–03），ROI 为左 IFG（主要）与双侧 PT / planum temporale（参照）。

主表示层为约 2/3 相对深度的标准化主层；最终层仅作三名被试 × IFG 的预设稳健性核查。Context Gain 拆为 Δr_local（r32−r8）、Δr_long（r128−r32）、Δr_total（r128−r8），主指标为 Δr_total。

本文档是完整技术规范与预注册实施依据，把工程拆成 8 个里程碑（M0–M7）。日常推进以配套的 `MILESTONE_EXECUTION_OVERVIEW.md` 为主；需要核对方法、边界条件、字段和验收细节时再查阅本文档。本文档在本次修订后冻结，不再继续扩展为 V5.0。

## 执行逻辑：先去风险，再放大

整条路径不按计划书章节顺序执行，而按技术风险和依赖关系执行。

M1 和 M2 是两条正交的去风险轨道，可以并行：M1 只证明“四个模型能在受控开窗下正确输出逐词表示”，不碰 fMRI；M2 只证明“词级特征到 BOLD 的对齐与编码管线正确”，不碰四个正式语言模型。两条轨道分别用状态重置测试、索引一致性检查和 English1000 / synthetic alignment test 独立验收。

两条轨道汇入 M3。M3 先进行不查看 outer held-out r 的盲态参数核查（M3a），确认 λ 网格、PCA、mask 和求解器均已最终冻结；随后用一个 Transformer 和一个状态模型各完成一条最小端到端竖切（M3b）。M3 通过后，M4 才扩展到完整正常条件与 time-shift 条件矩阵，M5 只负责预注册对比、置信区间与解释，避免在统计阶段重新进行大规模模型拟合。

## 三个信心检查点

1. M1：四模型状态重置、层位、窗口边界与 token/word 映射测试全部通过；AWD-LSTM 完成限时可行性审计。
2. M2：English1000 与参考实现达到预设数值一致性；synthetic alignment、故事边界和 FIR 边界测试通过。
3. M3：盲态参数核查完成并重新冻结后，Transformer 与状态模型两条竖切均能生成维度正确、无泄漏的 held-out 结果；time-shift 不出现明显的循环移位或标签泄漏迹象。

三个检查点全部通过后，后续主要是矩阵扩展、统计和呈现，不应再引入新的方法逻辑。

## 任务优先级与工期取舍

为避免工程级可复现基建挤压 8 周主实验，所有任务按以下三级管理：

- **P0｜实验有效性底线：必须完成。** 删除后可能造成静默错位、训练—测试泄漏、上下文失控、故事边界污染或主问题无法回答。P0 未完成，不得进入下一里程碑。
- **P1｜完整硕士论文交付：原则上完成。** 用于形成完整结果、稳健性分析和规范论文材料；如发生工期风险，应在保住全部 P0 后压缩 P1 的展示范围，而不是改变主分析。
- **P2｜发表级可复现性加分：工期紧时可简化。** 包括逐文件 SHA256、每次临时运行的完整 manifest、扩展审计索引和额外格式化报告。P2 可简化为“关键冻结物一次 hash + Git tag + 正式主实验运行记录”。

八周排期默认优先完成 P0 和必要 P1。M7 及 P2 不得阻塞 LeBel 主实验闭环。

## 毕业闭环定义

原则上的毕业闭环为：完成 M0–M6，包括 LeBel 三名被试、四模型、H={8,32,128}、标准化主层、左 IFG 与双侧 PT、三折故事级 CV、training-only scaler/PCA、统一 voxelwise Ridge、三类 Context Gain、H-specific 架构差值、配对 story bootstrap、逐 H 的 40 秒 time-shift 与 shifted Context Gain、IFG 最终层稳健性核查、>100s 共同评价 mask。

AWD-LSTM 是计划内的正式历史参照，不得在执行过程中默认删除。若其在预设风险预算内仍无法稳定接入，必须形成书面的 scope amendment，记录失败原因、已尝试方案、对论文题目与研究问题的影响，并经导师确认后，才能将正式范围改为核心三模型。未经正式变更，不得同时删除 LSTM 又声称完成原四模型计划。

M7（LPP）、Narratives、native state、完整 CCnorm、全皮层均不影响毕业闭环。

## 有效性纪律由结构强制执行

fold 划分、ROI 规则、上下文定义、层位、评价 mask、Ridge 选择规则、PCA 规则、time-shift 规则和统计 contrasts 均在查看正式 held-out 结果前冻结。最低审计证据（P0）为：

- 模型 revision / commit；

# 里程碑 0：地基、冻结与可复现骨架

## 周期与验收要求

- 周期：第 1 周。起止日期：待锚定。
- 前置依赖：无。

## 阶段目标

把所有“查看正式结果前必须确定”的研究规则固定下来，并搭好工程骨架。本阶段不产生 brain score，不运行正式四模型特征提取，不跑正式编码模型。

完成后，应固定并可追溯：

- 数据、故事与被试范围；
- outer fold；
- ROI 与 BOLD-only voxel inclusion rule；
- H 的精确定义和索引规则；
- 主层与最终层；
- W_common 的构建原则；
- scaler/PCA/Ridge 的训练与选择规则；
- >100s、FIR 和 time-shift 的有效时间点规则；
- repeatability 计算规则；
- 预注册 contrasts；
- 软件环境、随机种子与运行日志格式。

### 本阶段优先级

- **P0：**主配置、story/word 索引、fold、ROI、H 定义、PCA/Ridge/time-shift 规则、contrast 分层、Git freeze tag。
- **P1：**环境 lock、目录规范、正式运行 manifest 模板。
- **P2：**逐文件 SHA256、完整数据 hash 索引、所有临时运行的 manifest。

## 设计思路

所有参数从一个主配置入口读取，代码中不得散落魔数。数据级冻结信息落入 `frozen/`，运行环境落入 lock file，统计对比落入 `contrast_registry.yaml`。M3b、M4 和最终统计等正式运行必须写出包含 Git commit、配置 hash、数据版本与模型 revision 的 run manifest；smoke test 和临时调试运行可简化记录。

项目建立一份贯穿文本、token、TR、fold 和 QC 的统一索引。`word_index.parquet` 是所有条件共同的原始词事实表；模型自己的 subtoken span 另存为模型级 token map。后续任何模型特征、时间对齐或评分都通过这些索引关联，禁止依赖数组位置的隐式一致性。

H 表示目标词之前允许访问的原始历史词数，不包含目标词本身。因此 H=8 时实际模型输入为 8 个历史词 + 当前目标词，共 9 个原始词。使用 0-based 索引时，第一个拥有完整 H=128 历史的目标词为 i=128；等价的一号起始位置为第 129 个词。

## 预注册对比表与多重比较声明

`frozen/contrast_registry.yaml` 至少包含以下分层。只有 `confirmatory_primary` 属于确认性推断家族；其余均为探索性、描述性或稳健性结果。

```yaml
confirmatory_primary:
  family_name: ifg_main_layer_delta_total_architecture
  fwer_control: holm_bootstrap_pvalues_alpha_0.05
  contrasts:
    - rwkv_minus_pythia_delta_total_ifg_main
    - mamba_minus_pythia_delta_total_ifg_main

main_estimands_descriptive:
  - delta_total_pythia_ifg_main
  - delta_total_rwkv_ifg_main
  - delta_total_mamba_ifg_main

exploratory_rq1_same_context_architecture:
  - rwkv_minus_pythia_r8
  - rwkv_minus_pythia_r32
  - rwkv_minus_pythia_r128
  - mamba_minus_pythia_r8
  - mamba_minus_pythia_r32
  - mamba_minus_pythia_r128

secondary_exploratory:
  - delta_local_by_model
  - delta_long_by_model
  - ifg_vs_pt_descriptive_pattern
  - awd_lstm_within_model_context_curve

robustness:
  - registered_contrasts_on_final_layer_ifg
  - shifted_r_by_h
  - shifted_delta_local
  - shifted_delta_long
  - shifted_delta_total
```

**多重比较声明：**确认性结论仅限左 IFG、标准化主层下两项核心三模型 `Δr_total` 架构差值，并把二者视为同一个检验家族，使用 Holm 方法控制双侧 family-wise error rate（α=0.05；基于配对 bootstrap 分布得到检验概率）。所有 H-specific RQ1 差值、`Δr_local`、`Δr_long`、PT、AWD-LSTM、最终层和 time-shift 结果均报告效应量与未校正 95% bootstrap CI，但只作探索性、描述性或稳健性解释，不据此宣称新的确认性发现。

## 模块与目的

1. 配置加载与校验模块  
   目的：所有里程碑从单一配置读取参数；缺字段、非法值和互相冲突的设置直接报错。

2. Story manifest 生成器  
   目的：清点 UTS01–03 的独立故事与重复故事，输出故事名、时长、有效 TR 数、独立/重复标记和数据版本；逐文件 hash 属于 P2。

3. 统一 word index 生成器  
   目的：建立 story、word_id、原始词、onset、目标位置、TR、fold、>100s 标记、QC flag 等共同索引。

4. Fold 划分器  
   目的：按故事时长与有效 TR 数尽量均衡地生成固定三折，并排除重复故事。

5. ROI 与 BOLD-only voxel 规则冻结模块  
   目的：固定 atlas 标签、半球、最小 voxel 数、排除规则、NaN/零方差处理和 repeatability 方法。

6. 分析规则冻结模块  
   目的：固定 H、层位、PCA、Ridge、fold 汇总、time-shift、mask 与 contrasts。

7. 环境与运行清单模块  
   目的：锁定依赖版本，规定 run manifest、缓存目录与 hash 记录方式。

8. 资源预算模板  
   目的：记录 GPU、CPU、磁盘可用量，并为 M1 实测的每千词耗时和缓存大小预留预算字段。

## 验收标准

1. 冻结文件、环境 lock、Git tag、主配置 hash 与数据版本记录全部存在且互相一致；全量数据 hash 不作为进入 M1/M2 的硬门槛。

# 里程碑 1：模型特征与容量轨道

## 周期与验收要求

- 周期：第 2–3 周。起止日期：待锚定。
- 前置依赖：M0（配置、统一索引、故事清单、环境骨架）。
- 可与 M2 并行执行。
- 验收要求：在进入 M3 前完成。
- 验收方式：状态重置测试 + 窗口 off-by-one 测试 + token map 检查 + AWD-LSTM 限时审计 + 单故事 smoke test + 容量报告 + 模型审计表。

## 阶段目标

证明四个语言模型能在完全相同的原始词窗口上，正确输出目标词的逐词上下文化表示，并在同一次 forward 中保存主层与最终层。建立模型级 token map 和全量运行资源预算。本阶段不碰 fMRI 编码结果。

AWD-LSTM 是正式历史参照而不是默认可删项。其接入设 3 个工作日风险预算；若仍无法通过，必须形成 scope amendment 并经导师确认，不能直接把它降为 Future Work。

### 本阶段优先级

- **P0：**四模型加载、窗口与状态重置、off-by-one、token map、W_common、AWD-LSTM raw hidden/xxunk 审计、单故事 smoke test。
- **P1：**主层与最终层双缓存、全量运行容量试测。
- **P2：**格式化容量报告、扩展性能基准和完整缓存审计索引。

## 设计思路

四种模型共享：

\[
W_i(H)=[w_{i-H},\ldots,w_{i-1},w_i]
\]

其中 H 只统计目标词之前的历史词数，实际输入总词数为 H+1。不同 tokenizer 可产生不同 token 数，但原始词内容、目标位置和故事边界必须一致。

Transformer 不复用窗口外 KV cache；RWKV、Mamba、AWD-LSTM 每个窗口从零状态开始。状态重置必须通过正反单元测试证明：同一窗口重置后重复运行应一致；故意继承前窗状态时，输出应与冷启动结果不同。

W_common 由共同原始目标位置、完整 H=128 历史、有效 onset/TR 映射和四模型技术可输出性共同确定。AWD-LSTM 把词映射为 `xxunk` 仍视为成功输出，因此不因 `xxunk` 删除目标词，但必须报告 `xxunk` 比例和分布。

```python
def build_window(words, i, H):
    """H 是 preceding words 数；返回 H 个历史词 + 当前目标词。"""
    if i < H:
        raise ValueError("target does not have a complete history")
    window = words[i - H : i + 1]
    assert len(window) == H + 1
    assert window[-1] == words[i]
    return window


def extract_window_repr(model, words, i, H, layers, reset_state=True):
    window = build_window(words, i, H)
    if reset_state:
        model.reset_state()
    tokens, word_to_tok = tokenize_with_spans(window)
    hidden = model.forward(tokens, use_external_cache=False)
    target_last_token = word_to_tok[-1][-1]
    return {
        name: hidden[block_number_to_hidden_index(model, layer)][target_last_token]
        for name, layer in layers.items()
    }


def test_window_semantics(words):
    i = 128
    w8 = build_window(words, i, H=8)
    assert len(w8) == 9
    assert w8[0] == words[i - 8]
    assert words[i - 9] not in w8
    assert build_window(words, 128, H=128)[0] == words[0]


def test_state_reset(model, words, i, layers):
    a = extract_window_repr(model, words, i, 128, layers, reset_state=True)
    b = extract_window_repr(model, words, i, 128, layers, reset_state=True)
    assert allclose(a["main"], b["main"])

    c = inherit_previous_window_then_extract(model, words, i, 128, layers)
    assert not allclose(a["main"], c["main"])


def test_awd_lstm_layer(model, words):
    out = extract_window_repr(
        model, words, i=200, H=8,
        layers={"main": 2, "final": 3}
    )
    assert out["main"].shape[-1] == 1152
```

## AWD-LSTM 专项边界

1. **stateful 与层名**：显式清空每层 hidden/cell；提取第 2 recurrent layer 的 raw hidden，不取 decoder logits。
2. **词表覆盖**：`xxunk` 不导致 W_common 删除，但记录总体、按故事和按词频的 `xxunk` 比例。
3. **维度差异**：AWD-LSTM hidden=1152，核心模型 hidden=768；K=100 后维度统一，但只把 AWD-LSTM 解释为历史参照。
4. **正式变更规则**：三天风险预算结束后仍失败，提交 scope amendment；必要时同步修改题目、RQ、图表和结论边界。

## 阶段目标

M3 分成两个明确阶段：

- **M3a：盲态参数核查。** 只使用 outer training stories 和 inner validation，检查 λ 边界、PCA、求解器、维度与内存；不得生成、保存或展示 outer held-out r。
- **M3b：双路径竖切。** 在重新冻结后，用 UTS01 × IFG × H=32 × 主层 × 一个 outer fold，分别跑 M0 已冻结的 Pythia 与 Mamba，生成第一次正式 held-out r。Mamba 在此仅代表状态空间适配路径，不代表其研究地位高于 RWKV。

### 本阶段优先级

- **P0：**M3a 盲态边界核查、禁止架构间 inner-score 比较、重新冻结、Pythia+Mamba 双路径竖切、泄漏审计。
- **P1：**单条件 time-shift 技术诊断和规范 story-level 输出。
- **P2：**扩展性能日志与额外诊断图。

## 设计思路

M3a 若发现大量 λ 落在网格边界，只能依据 inner validation 扩展网格；修改后必须更新 `analysis_spec`、重新计算 hash 并打新的 freeze tag。M3a 只允许查看数值稳定性、边界命中率和求解器诊断，不得比较不同架构的 inner-validation 性能，也不得据此调整模型、层位、上下文、ROI、特征处理或统计方案。看过 outer held-out r 后不得再扩网格。

M3b 的 time-shift 只用于发现明显实现错误，例如循环回卷、故事串联、shift 前后预测完全相同或标签泄漏。单个被试、fold、模型中 shifted r 未明显降低，不作为项目硬失败条件；正式负控制结论由 M4 全矩阵和 M5 统计判断。

# 里程碑 5：预注册对比、统计与稳健性判定

## 周期与验收要求

- 周期：第 7 周。起止日期：待锚定。
- 前置依赖：M4（正常/shifted story-level 结果和最终层 IFG 结果）。
- 验收要求：在进入 M6 前完成。
- 验收方式：contrast registry 对照 + 配对索引检查 + CI 层级检查 + shifted Context Gain 检查 + 层位翻转规则检查。

## 阶段目标

对 M4 已生成的故事级结果进行预注册统计，完整回答：

- RQ1：相同 H 下核心三模型的 brain alignment 是否不同；
- RQ2：不同模型的 Context Gain 是否不同；
- RQ3：IFG 与 PT 是否呈现不同的描述性上下文模式；
- RQ4：主结论是否依赖主层而非最终层。

本阶段不再重跑语言模型、PCA 或 Ridge。

统计解释严格分层：两项左 IFG 主层 `Δr_total` 架构差值属于确认性家族；RQ1 的 H-specific 差值、`Δr_local`、`Δr_long`、PT、AWD-LSTM、最终层与 time-shift 均属于探索性、描述性或稳健性分析。探索性结果即使 95% CI 不跨 0，也不得单独表述为新的确认性发现。

### 本阶段优先级

- **P0：**两项确认性 `Δr_total` 对比、Holm 家族控制、配对 story bootstrap、shifted Context Gain、结论边界。
- **P1：**RQ1 H-specific 探索性差值、局部/长程增益、最终层、PT 与 repeatability。
- **P2：**额外未预注册可视化；不得升级为确认性结论。

## 设计思路

配对 story bootstrap 在每个 fold 内按故事有放回重抽；同一次重抽对所有模型、H、层位和 alignment condition 使用相同 story indices。先形成每个 fold 的加权 Fisher-z 汇总，再按冻结规则汇总 folds。权重使用每故事共同有效 TR 数，不把 TR 或 voxel 当作独立人类样本。

除三类 Context Gain 外，RQ1 必须直接计算每个 H 下的配对架构差值：

\[
D_{RWKV,H}=r_{RWKV,H}-r_{Pythia,H}
\]

\[
D_{Mamba,H}=r_{Mamba,H}-r_{Pythia,H}
\]

仅分别给三条曲线的 CI 不能替代直接差值 CI。

## 代码示例

```python
def paired_story_bootstrap(story_scores, registry, n_boot=1000, rng=None):
    """所有条件共用同一组 story 重抽索引。"""
    draws = []
    for _ in range(n_boot):
        sampled_by_fold = {
            fold: rng.choice(stories_in_fold(fold), size=len(stories_in_fold(fold)), replace=True)
            for fold in OUTER_FOLDS
        }
        r = aggregate_all_conditions(story_scores, sampled_by_fold)

        delta_local = {m: r[m][32] - r[m][8] for m in MODELS}
        delta_long = {m: r[m][128] - r[m][32] for m in MODELS}
        delta_total = {m: r[m][128] - r[m][8] for m in MODELS}

        rq1 = {
            ("rwkv", H): r["rwkv"][H] - r["pythia"][H]
            for H in (8, 32, 128)
        } | {
            ("mamba", H): r["mamba"][H] - r["pythia"][H]
            for H in (8, 32, 128)
        }

        gain_contrasts = {
            "rwkv_minus_pythia": delta_total["rwkv"] - delta_total["pythia"],
            "mamba_minus_pythia": delta_total["mamba"] - delta_total["pythia"],
        }
        draws.append((r, delta_local, delta_long, delta_total, rq1, gain_contrasts))

    return percentile_ci(draws, 0.025, 0.975)
```

## 模块与目的

1. Contrast registry 执行模块  
   目的：逐项计算已冻结的 RQ1、Context Gain、shifted 和层位对比。

2. 配对 story bootstrap 模块  
   目的：在故事级、被试内和 fold 内进行成对不确定性估计。

3. RQ1 H-specific 差值模块  
   目的：直接检验相同上下文下 RWKV/Mamba 相对 Pythia 的 r 差值。

4. Context Gain 模块  
   目的：计算 Δr_local、Δr_long、Δr_total 和相对 Pythia 的增益差值。

5. Shifted control 模块  
   目的：从 M4 shifted 结果计算 shifted r 和 shifted Context Gain，不重新拟合模型。

6. 层位稳健性模块  
   目的：在 IFG 上执行预注册 contrasts 的主层/最终层比较和实质翻转规则。

7. Repeatability 与 ROI 解释模块  
   目的：将 IFG/PT repeatability 与效应和 CI 并列报告。

## 交付物

1. 每被试、每 ROI、每模型的 r8/r32/r128 与 95% CI。
2. 每个 H 下 RWKV−Pythia、Mamba−Pythia 的 r 差值与 95% CI。
3. 三类 Context Gain 与 95% CI。
4. 两项确认性 Δr_total 架构差值、95% CI、bootstrap p 值与 Holm 校正结果。
5. 正常与 shifted 的 r、Δr_local、Δr_long、Δr_total。
6. IFG 主层与最终层的预注册对比及实质翻转判定。
7. IFG/PT repeatability 与 ROI 解释表。
8. 完整统计结果文件和 contrast execution log。

## 任务拆解

1. 故事级结果核查
   - 检查每个故事在所有成对条件下均存在。
   - 核查共同 mask 和有效 TR 权重。

2. 执行 RQ1 contrasts
   - 对 H=8/32/128 分别计算核心模型相对 Pythia 的配对差值 CI。

3. 执行 Context Gain contrasts
   - 计算三类 Δr 和 Δr_total 架构差值。
   - 对两项确认性 Δr_total 架构差值计算 bootstrap p 值并执行 Holm 家族校正。

4. 配对 bootstrap
   - fold 内故事重抽；全部条件共用 indices；1000 次。
   - 输出点估计、95% CI 和随机种子。

5. Shifted controls
   - 从 M4 shifted 结果计算 shifted r 与三类 shifted Δr。
   - 若 shifted 模式稳定复制主要 Architecture × Context 结果，停止机制解释并排查漂移/位置伪影。

6. 层位稳健性
   - 对同一预注册 contrast 比较主层与最终层 CI。
   - 只有两层 CI 分别完全位于 0 的相反两侧，才称实质翻转。

7. ROI/repeatability 解释
   - PT 近零仅在 repeatability 可接受、基础对齐可检出、shift 后下降且 CI 足够窄时，谨慎表述为区域特异性支持；否则写“未检测到可靠效应”。

## 验收标准

1. `contrast_registry.yaml` 中所有预注册条目均有结果或明确缺失原因，且确认性/探索性/稳健性标签未改变。
2. RQ1 由 H-specific 配对差值直接回答，而不是只比较三条独立 CI。
3. bootstrap 以 story 为重抽单位，不把 voxel/TR 当作独立人类样本。
4. 所有模型、H、层位和 normal/shifted 条件使用相同重抽 indices。
5. shifted r 和 shifted Context Gain 均已计算；异常模式已触发解释限制或排查。
6. 层位翻转按相反非零 CI 规则判定，不以点估计排名替代。
7. IFG/PT 解释结合 repeatability、基础 r、CI 宽度和 shift 结果；只有两项确认性主对比可下确认性结论。

## 明确不做

1. 不做未预注册的复杂 mixed-effects 模型作为主结论。
2. 不计算完整 CCnorm 或正式 noise ceiling。
3. 不在 bootstrap 中重跑语言模型、PCA 或 Ridge。
4. 不据模型表现选择 voxel。
5. 不把 AWD-LSTM 绝对 r 纳入核心三模型强排名。

1. 五图两表和 QC 表完整，且每张图/表对应明确 RQ 或控制目的。
2. Results 通过直接 contrasts 回答 RQ1–RQ4。
3. Methods 与代码实现、配置和 run manifest 一致。
4. AWD-LSTM 仅作历史参照，图表和文字均未用于核心强排名。
5. 只使用采样点措辞，不声称精确饱和点或函数形式。
6. 结论限定为 checkpoint-level、被试内、间接机制证据。
7. 可复现包能定位每项结果的 Git tag、配置 hash、数据 hash、模型 revision 和运行命令。

## 明确不做

1. 不在 M6 重新选择分析规则或统计 contrasts。
2. 不根据图表回头修改冻结的 fold、ROI、层位、mask 或 λ。
3. 不让 LPP/Narratives 等可选分析阻塞毕业闭环。
4. 不用 AWD-LSTM 绝对 r 进行核心架构排名。

# 里程碑 7：条件性 LPP-English 复现

## 周期与验收要求

- 周期：条件性，无固定周。
- 前置依赖：M6 完成。
- 验收要求：非毕业必需；开始前必须完成独立的 M7a 冻结。
- 验收方式：M7a 配置冻结检查 + 数据适配检查 + 方向一致性检查。

## 阶段目标

在 LPP-English 上进行缩减规模的方向性外部复现：H ∈ {8,128}、左 IFG、主层。目标是检验 LeBel 中预注册的 Architecture × Context 方向是否在独立数据集上方向一致，而不是追求完整复制或绝对 r 对齐。

## M7a：启动前独立冻结

不得直接沿用 LeBel 中未必适合 LPP 的设置。开始任何 LPP brain score 前，必须单独冻结：

- 纳入被试与排除标准；
- split unit（章节、run 或故事）与 fold 数；
- 重复/共享文本的处理；
- LPP 的 TR、预处理和 ROI 定义；
- H={8,128} 的共同目标与评分 mask；
- scaler/PCA/Ridge 是否完全沿用主实验；
- 是否执行 40s time-shift；
- 方向一致性的操作定义；
- 缺失被试或低可靠性时的处理。

这些设置写入 `frozen_lpp/lpp_analysis_spec.yaml`，提交新 Git tag 后才进入 M7b。

## 方向一致性预定义

方向一致性至少应在启动前选择一种明确规则，例如：

- 对每个核心模型，LeBel 与 LPP 的 Δr_total 点估计符号一致；
- 或核心对比（RWKV−Pythia、Mamba−Pythia 的 Δr_total）方向一致；
- 若 CI 过宽或跨 0，结论为“不确定”，不得强行判为复制失败或成功。

实际采用哪一条必须在 M7a 冻结，不能在看 LPP 结果后选择。

### 本阶段优先级

- **P0：**无；M7 不属于毕业闭环。
- **P1：**仅在 M0–M6 完成且工期充足时，先冻结 LPP 被试、CV、mask、统计和方向一致性标准，再运行方向性复现。
- **P2：**额外数据集或扩展 ROI，不进入本项目正式范围。

## 设计思路

LPP 与 LeBel 的 BIDS 结构、预处理、TR、被试数量和 ROI 空间可能不同，因此只复用语言模型特征提取和已验证的无泄漏原则，不假设 fMRI 侧可直接复制。缩减到 H={8,128}、左 IFG 和主层，以最小成本保留 Δr_total 方向性证据。

## 模块与目的

1. M7a 配置与预注册模块  
   目的：冻结被试、CV、ROI、mask、统计与方向一致性定义。

2. LPP 数据适配模块  
   目的：处理 BIDS、TR、预处理和文本/扫描对应关系。

3. LPP ROI 模块  
   目的：在 LPP 空间中定义与主实验概念对应的左 IFG。

4. 特征复用模块  
   目的：复用 M1 的四模型主层、开窗和状态重置逻辑。

5. LPP 无泄漏编码模块  
   目的：按冻结的 split unit 执行 training-only scaler/PCA/Ridge。

6. 方向性统计模块  
   目的：计算 H=8/128 的 r、Δr_total 和预定义方向一致性。

## 交付物

1. `frozen_lpp/lpp_analysis_spec.yaml` 和独立 freeze tag。
2. LPP 数据、文本、TR 和 ROI 适配报告。
3. LPP 的 CV 与评分 mask 说明。
4. 四模型 H={8,128} 的 held-out r 与 Δr_total。
5. 按预定义规则给出的“一致 / 不一致 / 不确定”结论。
6. 若执行 time-shift：对应 shifted r 与 shifted Δr_total。

## 任务拆解

1. M7a 冻结
   - 清点数据并确定被试、split unit、fold、ROI、mask 和统计规则。
   - 冻结方向一致性定义并打 tag。

2. 适配 LPP 数据
   - 对齐文本、onset、TR、BOLD 与 ROI。
   - 检查故事/章节边界和有效时间点。

3. 复用模型特征
   - 提取四模型主层 H={8,128} 表示。
   - 建立 LPP 独立 W_common 和 token map。

4. 执行无泄漏编码
   - 所有变换仅在 LPP training split 内拟合。

5. 方向性判断
   - 计算 r、Δr_total 和预注册核心 contrasts。
   - 依据 M7a 规则报告一致、不一致或不确定。

## 验收标准

1. 所有 LPP 分析设置在第一次 brain score 前冻结。
2. split unit、fold、被试、ROI、mask 和统计规则明确且可追溯。
3. 无跨章节/run 泄漏，training-only 变换成立。
4. 四模型 H={8,128} 的结果完整，或缺失有书面原因。
5. 方向一致性严格按预定义规则判断。
6. LPP 结果不改变已经完成的 LeBel 主实验冻结规则和毕业闭环。

## 明确不做

1. 不直接复制 LeBel 的 fMRI 管线而不验证适用性。
2. 不扩展到 H=32、PT、最终层或全皮层。
3. 不与 LeBel 的绝对 r 直接比较。
4. 不根据 LPP 结果重新定义“方向一致”。
5. 不让 M7 阻塞 M0–M6 的毕业闭环。
6. 不执行 Narratives（仅 Future Work）。
